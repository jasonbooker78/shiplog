# Shiplog

A lightweight kanban board for agentic design teams. Import tasks from an LLM-generated PRD, manage your backlog, and export shipped work to Jira — without slowing down.

![Shiplog board](public/shiplog-logo.svg)

---

## What it does

- **Import** — paste a JSON task list generated by Claude or any LLM directly onto your board. A built-in prompt helper reformats any existing PRD into the required schema.
- **Kanban board** — three columns: Backlog, In Progress, Shipped. Drag tasks between columns, filter by feature area, and add notes inline.
- **Task management** — create, edit, and delete tasks with title, description, priority, feature area, status, and notes.
- **Jira CSV export** — export all shipped tasks as a Jira-compatible CSV with one click.
- **Sharing** — invite collaborators to co-edit a board, or copy a view-only link for stakeholder access (Phase 2).

## Stack

| Layer | Choice |
|---|---|
| Framework | React 18 |
| Build | Vite |
| Routing | React Router v6 |
| Icons | Lucide React |
| Fonts | Syne, IBM Plex Mono (Google Fonts) |
| Styling | CSS custom properties (design tokens) |
| State | Local `useState` (Phase 1) |

No external state library. No backend in Phase 1 — all data lives in React state.

## Design tokens

All colors are CSS custom properties defined in `src/index.css`. Never hardcode hex values in components.

| Token | Usage |
|---|---|
| `--bg-base` | page background (warm off-white) |
| `--bg-surface` | card / input backgrounds |
| `--bg-elevated` | header, modal backgrounds |
| `--accent` | primary CTA, active states |
| `--text-primary` | body text |
| `--text-secondary` | supporting text |
| `--text-dim` | labels, metadata |
| `--border-subtle` / `--border-mid` | dividers, input borders |
